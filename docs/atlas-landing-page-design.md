## 目的

本ドキュメントは、Nexcon Lab 公式サイトに **Atlas Health Platform 専用の紹介ページ（ランディングページ）** を追加するための設計書である。

**ページのゴール:**

> Atlas を初めて見る **中小企業の経営者** が、  
> 読み終えたときに **「うちの会社にも必要かもしれない」** と感じること。

**設計方針:**

- 技術用語（Telemetry, API, Agent 等）は **裏側** に留め、表側は **業務言語** で語る
- 「PC が壊れたら困る」→「業務が止まる前に気づける」へ変換
- Pilot Edition v0.4 の **実装済み機能のみ** を正確に訴求（過剰約束しない）
- トップページ（`/`）の Products カードから **詳細ページへ誘導** する構造

**本フェーズで行わないこと:** コード実装、デザイン変更、Vercel 設定変更

## 1. ページ目的

### 1.1  Primary Goal

| 観点 | 内容 |
|------|------|
| **Who** | 経営者・役員・総務責任者（IT 専任者不在） |
| **What** | Atlas が「PC の予防医療」であることを理解してもらう |
| **Why now** | ディスク不足・Outlook 障害が **突然の業務停止** につながるリスクを認識 |

- Nexcon Lab の **第 1 製品** として Atlas のブランドを確立

- LINE予約システムとの **製品群の差別化**（社内 IT vs 対外予約）

| KPI | 目安 |

| ページ滞在時間 | 2 分以上 |
| CTA クリック率 | 3% 以上 |

### 2.2 ターゲット条件（要件より）

### 3.2 各セクション詳細

#### 1. Hero

| 要素 | 内容 |
|------|------|
| 製品ロゴ | Atlas / Health Platform（Pilot Edition バッジ） |
| メインコピー | §4 から選定 |
| サブコピー | IT 不在の中小企業向け PC 健康管理 |
| CTA _primary | 無料相談・デモ依頼 |
| CTA _secondary | 資料請求 |
| ビジュアル | Dashboard スクリーンショット（PC健康スコア） |

#### 2. 課題提起

**見出し例:** 「PC のトラブル、気づいたときには業務が止まっていませんか？」

| 課題 | 業務影響（コピー案） |
|------|----------------------|
| ディスク満杯 | 印刷できない、更新できない、Outlook が開かない |
| PC 状態不明 | 誰が危ないかわからない、口頭・Excel 管理 |
| 故障後対応 | 代表・総務の時間が奪われる |
| 修復が怖い | 詳しい人に頼むしかない |

#### 3. Atlas の解決方法

**3 ステップ構成:**

```
監視する  →  わかる  →  安全に直す
   │            │            │
 Agent      Finding      承認付き修復
 60秒更新    業務影響表示   Guardrails
```

技術名は補足テキストまたはツールチップに格下げ。

#### 4. 機能紹介

| 機能 | 見出し（業務語） | 実装状態 |
|------|------------------|----------|
| PC健康スコア | 全社 PC を 0–100 で一目把握 | ◎ v0.4 |
| 自動診断 | 容量不足・Outlook リスクを自動検知 | ◎ v0.4 |
| 安全修復 | 承認後に限定修復（temp_cleanup） | ○ v1 |
| 監査ログ | 誰がいつ修復したか記録 | ◎ v0.4 |

#### 5. Dashboard 紹介

- スクリーンショット 4 枚（§6）
- 各画像に **1 行キャプション**（業務ベネフィット）
- モバイルではカルーセル or 縦積み

#### 6. 導入メリット

| メリット | 説明 |
|----------|------|
| **業務継続** | 停止前に予兆をキャッチ |
| **予防保守** | 故障後の火消しから予防へ |
| **見える化** | 経営会議で PC 状態を説明可能 |
| **IT 不在でも回る** | 総務・事務が Dashboard で対応 |
| **データは社内** | Pilot はオンプレ・localhost 完結 |

#### 7. 導入対象

- PC **5〜50 台**
- Windows 10 / 11 事務 PC
- Outlook / Microsoft 365 利用
- 製造・食品製造・地方 SMB
- IT 専任者不在 or 総務兼 IT

**不向き:** 大規模 Enterprise、Mac のみ環境

#### 8. 今後のロードマップ

## 5. Atlas の価値提案

### 5.1 価値の 3 本柱（技術ではなく業務）

```
        ┌─────────────┐
        │  業務継続    │  … 止まる前に気づく
        └──────┬──────┘
               │
    ┌──────────┼──────────┐
    │          │          │
┌───┴───┐ ┌───┴───┐ ┌───┴───┐
│予防保守│ │見える化│ │安全対応│
└───────┘ └───────┘ └───────┘
```

### 5.2 業務継続

| 項目 | 内容 |
|------|------|
| **訴求** | 「PC トラブル = 売上・生産・連絡が止まる」 |
| **Atlas** | 60 秒間隔の監視、Finding による早期アラート |
| **言い換え** | 「Agent」「Telemetry」→ **「常に見守る仕組み」** |

### 5.3 予防保守

| 項目 | 内容 |
|------|------|
| **訴求** | 故障後の駆けつけコスト > 予防のコスト |
| **Atlas** | ディスク・Outlook の予兆検知、修復 v1 |
| **言い換え** | 「ルールエンジン」→ **「自動で異常を見つける」** |

### 5.4 見える化

| 項目 | 内容 |
|------|------|
| **訴求** | 経営者が「IT 状態」を会議で説明できない |
| **Atlas** | PC健康スコア 0–100、KPI、Action 一覧 |
| **言い換え** | 「Dashboard KPI」→ **「全社 PC の健康診断結果」** |

### 5.5 避けるべき訴求（ターゲット不匹配）

- RMM / MDM 完全代替
- 全自動修復（現状は限定スクリプト + 承認フロー）
- クラウド必須（Pilot はオンプレ）
